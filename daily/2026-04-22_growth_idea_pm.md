## 1. AIエージェント役員の自律成長方法（PM視点）

### 午前との違い

午前は **内部記録ループ**（仕事実行 → growth_log → playbook蓄積）を提案した。  
午後は **外部情報の能動的インプット** と **役員間クロスフィードバック** に焦点を当てる。

---

### 2026年のAIエージェントが実装している「自律学習の2本柱」

**柱1: 永続記憶（Persistent Memory）**  
Mem0・MemoryLakeなどが2026年にプロダクション実用化。  
エージェントは「ベクトルメモリ」と「グラフメモリ」を組み合わせ、過去の経験から関係性・パターンを自動抽出・更新する。  
→ AI Companyへの応用: 各役員が **週次で外部情報をインプットし、playbook.mdに「市場観察ノート」として蓄積** する。

**柱2: 自己改善（Self-Improvement）**  
MetaのHyperagentsやHermes Agentが実装。  
「行動結果を評価 → 改善案を生成 → 次回の行動に反映」するメタ認知ループ。  
→ AI Companyへの応用: **役員間の相互フィードバック（週1回・5分）** で代替する。

---

### 各役員の「外部インプット」担当領域と実装方法

#### CoS（戦略参謀）
**担当領域:** 競合戦略・市場価格帯・補助金制度の変化  
**実装方法:**
- 毎週水曜: 競合Web制作会社5社のサービスページを比較し「差別化点」をplaybook.mdに追記
- 毎月1日: 中小企業庁・愛知県の補助金情報を確認し、活用可能な新制度をCOOに共有

#### CMO（集客・営業）
**担当領域:** バズコンテンツのパターン・競合の営業文言・SNSアルゴリズムの変化  
**実装方法:**
- 毎日: X（Twitter）のAI・副業・中小企業関連の上位バズ投稿3件をgrowth_logに記録
- 週1: 競合のキャッチコピー・メール件名を収集して比較メモを作成
- 2026年知見: **Xアルゴリズムは「保存数 > いいね > RT」の順で評価。「まとめ型」投稿が最もバズりやすい**

#### CPO（商品設計）
**担当領域:** 顧客ニーズの変化・競合の価格・サービス構成  
**実装方法:**
- 週1: ランサーズ・クラウドワークスのWeb制作案件「要望コメント」からニーズを10件抽出
- 月1: noteの有料記事「売れ筋ランキング」Top10を分析し、商品設計に反映

#### COO（業務・管理）
**担当領域:** 業務自動化ツールの最新情報・SOP標準事例  
**実装方法:**
- 週1: Make.com / Zapierの新テンプレートから「AI Companyの業務に応用可能なもの」を1つ選定
- 月1: 同規模（1人社長 + AI）の成功事例を1件リサーチしてCOO playbookに追記

#### CTO（技術・AI）
**担当領域:** AIフレームワーク・モデルの最新動向  
**実装方法:**
- 週1: GitHubトレンドのAI関連リポジトリTop5を確認 → AI Companyへの応用可能性を評価しplaybookに追記
- 週1: Anthropic / OpenAIのリリースノートを確認 → 既存プロンプトの改善点を記録
- 2026年知見: **グラフメモリ（Mem0 graph-enhanced）がプロダクション実用段階に到達。役員間情報連携に応用余地あり**

---

### 役員間クロスフィードバック（週1回・月曜朝・5分）

```
フォーマット（各役員が1行ずつ）:
CoS → CMO: 「先週の営業・集客で改善すべき点1つ」
CMO → CPO: 「先週の顧客反応で商品に反映すべき点1つ」
CPO → COO: 「商品設計で業務フロー変更が必要な点1つ」
COO → CTO: 「自動化したい業務1つ」
CTO → CoS: 「技術面から見た戦略の改善点1つ」
```

役員間の知識伝達によりサイロ化を防ぎ、「部門横断的な自律改善サイクル」が生まれる。

## 2. AI Companyだけで収益を発生させるアイデア（PM）

### アイデア: 「AI陰陽師LINEBot」で24時間無人占いサービスを立ち上げる

**午前との違い:**  
午前は「AI自動営業エンジン」（能動的アウトバウンド営業）。  
午後は「受け身で入ってくる収益」（インバウンド・自動課金）。

**背景・根拠:**
- 占い・スピリチュアル市場は国内年間5,000億円超（2026年）
- LINE公式アカウントは月額2,000円〜で開設可能
- Claude APIを組み込めば占い師不要・24時間自動応答が実現可能
- 陰陽師・占い事業はAI Companyの7事業として既に計画済み
- 社長の稼働ゼロ、AI Companyの役員だけで構築・運用が完結

---

### 実行可能な3ステップアクションプラン

**Step 1（今週中）: 最小構成のLINE占いBotを作る**
- CTO: Claude API + LINE Messaging APIで「生年月日入力 → 今日の運勢を返す」Botを構築
- CPO: 占いメニュー設計（①今日の運勢 ②相性占い ③仕事運の3種）
- 使用技術: Anthropic Claude API + Python（Webhook）+ LINE公式アカウント
- コスト: LINE基本料 0〜2,000円/月 + Claude API使用料（数千円/月）
- 期間: 3日で最小構成の完成が可能

**Step 2（来週）: 有料メニューと決済を追加する**
- CTO: Stripe決済を実装（LINE Pay不要でシンプル化）
- CPO: 無料プラン（今日の運勢）+ 有料プラン（詳細鑑定 980円）の設計
- CMO: XとInstagramで「無料体験キャンペーン」を投稿して初期ユーザーを獲得
- 目標: 2週間でユーザー100人・有料転換10%（10人）

**Step 3（来月〜）: サブスク化と口コミ拡散**
- CPO: 月額980円の「毎日占いサブスク」を追加
- CMO: 友達紹介で1ヶ月無料の紹介プログラムで口コミ拡散を設計
- COO: 決済・解約・継続の全プロセスを自動化
- 社長の稼働: 月1回の内容チェックのみ

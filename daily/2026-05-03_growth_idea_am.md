## 1. AIエージェント役員の自律的成長方法

### 前提：現状の課題
全役員の `growth_log.md` に記録がゼロ。playbook の「蓄積されたノウハウ」も空。  
仕組みはあるが、**学習サイクルが回っていない**のが最大のボトルネック。

---

### 2026年最新トレンドを踏まえた成長フレーム

世界では以下が標準になっている（2026年時点）：
- **MCP（Model Context Protocol）**: Anthropicが開発しLinux財団に寄贈。ツール・データ接続の標準規格。AIエージェントが外部ツールを扱う共通インターフェース。
- **LangGraph v1.0**: 状態管理つきの自律ワークフロー構築の事実上の標準。ループ・分岐・ロールバックが可能。
- **Agent2Agent (A2A)**: Googleが提唱。複数AIエージェント間の連携プロトコル。役員間自律連携に直結。
- Gartnerによれば2026年時点で企業アプリの40%以上にAIエージェントが組み込まれている。

### 各役員の自律成長アクション

#### CoS（戦略参謀）
**現状の弱点**: 市場調査を毎回ゼロから実施。過去の調査結果が積み上がっていない。

**自律成長メソッド**:
1. **毎回の仕事前**: `growth_log.md` + `playbook.md` の「社長の判断パターン」を読み直す（5分）
2. **仕事後**: growth_logに「結論 / 根拠 / 学び」の3行を必ず追記。月2回以上同じパターンが出たらplaybookに昇格
3. **週次**: WebSearchで競合・市場トレンドを収集 → 「売上直結の情報」だけに絞り込み → CMO・CPOに共有

**playbook更新提案** (追記内容):
```
### 市場情報の収集周期
- 週1回: AI研修・Web制作市場の競合動向（検索ワード: 「AI研修 中小企業 2026」「ホームページ制作 補助金」）
- 月1回: 補助金制度の改正情報（デジタル化・AI導入補助金の公募状況）
```

#### CPO（商品設計）
**現状の弱点**: 商品ラインナップはあるが、価格の根拠・競合比較が記録されていない。

**自律成長メソッド**:

2. **新商品検討時**: 「30秒で説明できるか？」チェックを仕事前に必ず実施
3. **成約・不成約の記録**: 「なぜ選ばれたか / なぜ選ばれなかったか」を1行でgrowth_logに記録

**playbook更新提案** (追記内容):
```
### 価格改定のトリガー
- 競合が値下げ or 新機能追加 → 即座にCPO評価を実施
- AI Speed Web: 同業他社の月額相場が9,800円を下回ったら差別化ポイントを強化する
### 2026年の競合相場（初回調査）
- Web制作サブスク: 月8,000〜15,000円が主流

```

#### CMO（集客・営業）
**現状の弱点**: SNS投稿・メールテンプレートはあるが、効果測定→改善のループが未構築。

**自律成長メソッド**:
1. **週次**: X投稿のエンゲージメント上位3件の共通点を抽出 → `x_viral_generation_system.md` に追記
2. **月次**: 集客ファネルのボトルネック（認知/興味/比較/問い合わせ）をどこで詰まっているか確認
3. **営業メール送付後**: 開封率・返信率を記録。件名パターンごとに成績を比較し改善

**playbook更新提案** (追記内容):
```
### バズパターン（2026年5月時点）
- 「AI × 中小企業の具体的な数字」投稿が最も拡散しやすい

- スレッド形式（2〜5ツイート連投）はリーチ1.5〜2倍になる傾向
```

## 2. AI Companyだけで収益を発生させるアイデア

### アイデア: **AI Speed Web 完全自動営業パイプライン**

**概要**: AIエージェントが建設業・製造業・飲食店のWebサイトを自動診断し、パーソナライズされた営業提案書を生成・送信。人間の介在ゼロで問い合わせ → 月額契約獲得まで完結させる。

**なぜ今これか**:
- CTO製の `hp_checker.py`（HP品質診断）と `lead_verifier.py`（リード精査）がすでに存在
- CMOの `collection_workflow.md` に営業ファネルが設計済み
- 今月の最重要テーマ「AI Speed Web事業の立ち上げ」に直結
- 対象業種（建設・製造・飲食）・エリア（豊明→名古屋南部）が既に決定済み
- デジタル化・AI導入補助金2026で「AIエージェント活用ツール」が補助対象に認定 → 顧客側も補助金で導入コストをカバーできる

---

### 実行可能な3ステップアクションプラン

**STEP 1（今週中）: リード自動収集 + HP診断エージェントの結合**
- `lead_verifier.py` で豊明市の建設業・製造業・飲食店のWebサイトURLを自動収集（Googleマップ API or スクレイピング）
- `hp_checker.py` で各サイトを自動診断し「改善スコア」を算出
- 改善スコア低い順にリスト化（優先度高 = 提案価値が高い）
- 成果物: leads_豊明_2026-05.csv（スコア・業種・URL・連絡先付き）

**STEP 2（今週〜来週）: パーソナライズ提案書の自動生成**
- Claude APIを使い、hp_checker.pyの診断結果をもとに業種別・課題別の営業メール本文を自動生成
- 例: 「御社のWebサイトはスマホ表示が未対応です。建設業のお客様の73%はスマホで検索しています。月9,800円で対応済みのサイトに刷新できます」
- CMOのメールテンプレート（件名・本文・CTA）を変数化して自動挿入
- 成果物: 業種別メールテンプレート × 3種（建設・製造・飲食）

**STEP 3（来週〜月末）: 自動送信 + 返信トラッキング**
- リスト上位20社に自動メール送信（Gmailスケジュール送信 or SendGrid API）
- 返信があったらCOOがタスク化 → 社長が商談（ここだけ人間）
- 商談後の契約手続き: テンプレート化して半自動化
- 成果物: 自動送信済みログ + 返信率レポート（weekly）
